# 架构设计

## 系统总览

Herald 由三个组件构成，通过 WebSocket 连接：

```
┌─────────────────────────────────────────────────────────────┐
│                       整体架构                               │
│                                                             │
│  手机浏览器                服务器                 Mac          │
│  ┌────────┐    WSS    ┌──────────┐    WSS    ┌──────────┐  │
│  │ Web UI │◄─────────►│ 中继服务  │◄─────────►│ 菜单栏App │  │
│  └────────┘   E2EE    └──────────┘   E2EE    └────┬─────┘  │
│                       (只转发密文)              │         │
│                                            Accessibility   │
│                                               API          │
│                                                │           │
│                                         ┌──────▼─────┐     │
│                                         │ Terminal    │     │
│                                         │ (claude CLI)│     │
│                                         └────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

## 组件职责

### Mac 菜单栏应用 (`apps/mac`)

核心组件，运行在用户的 Mac 上：

```
┌─────────────────────────────────────┐
│           Mac App 内部架构           │
│                                     │
│  ┌───────────────┐                  │
│  │ ProcessMonitor │ 每 2 秒扫描      │
│  │ 发现 claude    │ claude 进程      │
│  │ 进程           │                  │
│  └───────┬───────┘                  │
│          │ PID                      │
│  ┌───────▼───────┐                  │
│  │ WindowMapper  │ PID → 终端窗口    │
│  │ 进程→窗口映射   │ AXUIElement     │
│  └───────┬───────┘                  │
│          │                          │
│  ┌───────▼───────┐                  │
│  │ TerminalReader│ 读取窗口文本内容   │
│  │ 终端内容读取    │ 检测内容变化      │
│  └───────┬───────┘                  │
│          │                          │
│  ┌───────▼───────┐                  │
│  │ OutputParser  │ 识别消息/diff/    │
│  │ 输出解析器     │ 工具调用/确认请求  │
│  └───────┬───────┘                  │
│          │ 结构化事件                 │
│  ┌───────▼───────┐                  │
│  │ RelayClient   │ WSS 连接中继服务器 │
│  │ 中继连接       │ 发送加密数据       │
│  └───────────────┘                  │
│                                     │
│  ┌───────────────┐                  │
│  │ InputHandler  │ 收到远程指令时     │
│  │ 输入处理       │ → 写入终端窗口    │
│  └───────────────┘                  │
└─────────────────────────────────────┘
```

**模块说明：**

| 模块 | 职责 | 关键 API |
|------|------|----------|
| ProcessMonitor | 定时扫描 `claude` 进程，维护活跃会话列表 | `NSRunningApplication`, `Process` |
| WindowMapper | 将进程 PID 映射到对应的终端窗口 | `AXUIElementCreateApplication`, `CGWindowListCopyWindowInfo` |
| TerminalReader | 通过 Accessibility API 读取终端窗口的文本内容 | `AXUIElementCopyAttributeValue` |
| OutputParser | 将原始终端文本解析为结构化事件 | 正则匹配 + 状态机 |
| RelayClient | 管理与中继服务器的 WebSocket 连接 | `URLSessionWebSocketTask` |
| InputHandler | 接收远程指令，通过 Accessibility API 写入终端 | `CGEvent`, `AXUIElementPerformAction` |

### 中继服务器 (`apps/server`)

极简的消息转发层，不处理业务逻辑：

```
┌─────────────────────────────────┐
│        中继服务器内部架构          │
│                                 │
│  ┌─────────────┐               │
│  │ HTTP Server │ 静态页面托管    │
│  │ (Hono)      │ 健康检查       │
│  └─────────────┘               │
│                                 │
│  ┌─────────────┐               │
│  │ WS Manager  │ 管理连接       │
│  │ 连接管理     │ Mac ↔ Phone   │
│  └──────┬──────┘               │
│         │                      │
│  ┌──────▼──────┐               │
│  │ PairService │ 配对码生成     │
│  │ 配对服务     │ 设备绑定       │
│  └──────┬──────┘               │
│         │                      │
│  ┌──────▼──────┐               │
│  │ AuthMiddle  │ JWT 验证       │
│  │ 认证中间件   │ 设备鉴权       │
│  └─────────────┘               │
└─────────────────────────────────┘
```

**设计原则：**

- 服务器只做转发，不解密、不存储消息内容
- 仅存储：设备 ID、公钥、配对关系
- 无状态设计，可水平扩展

## 数据流

### 1. Mac → 手机（查看输出）

```
TerminalReader 检测到文本变化
    ↓
OutputParser 解析为结构化事件
    ↓
CryptoLayer 端到端加密
    ↓
RelayClient 通过 WSS 发送到服务器
    ↓
服务器原样转发密文到手机
    ↓
Web CryptoLayer 解密
    ↓
对应渲染组件展示
```

### 2. 手机 → Mac（发送指令）

```
用户在 Web UI 输入文字或点击按钮
    ↓
CryptoLayer 端到端加密
    ↓
WSS 发送到服务器
    ↓
服务器原样转发密文到 Mac
    ↓
Mac CryptoLayer 解密
    ↓
安全检查（危险命令拦截）
    ↓
Mac 端确认弹窗（可选，高危指令必弹）
    ↓
InputHandler 通过 Accessibility API 写入终端
```

## 会话生命周期

```
        ┌─────────┐
        │ 已发现   │  ProcessMonitor 检测到新 claude 进程
        └────┬────┘
             │ 自动关联终端窗口
        ┌────▼────┐
        │ 已连接   │  WindowMapper 成功映射
        └────┬────┘
             │ 开始推送输出
        ┌────▼────┐
        │ 活跃中   │  正常运行，双向通信
        └────┬────┘
             │ 进程退出或窗口关闭
        ┌────▼────┐
        │ 已结束   │  保留最后输出快照
        └─────────┘
```

## 性能考量

| 指标 | 目标 | 实现方式 |
|------|------|----------|
| 终端扫描频率 | 2 秒 | ProcessMonitor 定时器 |
| 内容变化检测 | 500ms | TerminalReader diff 比对 |
| 端到端延迟 | < 200ms | WSS 长连接，无轮询 |
| Mac 端 CPU | < 3% | 仅在内容变化时处理 |
| 服务器内存 | < 50MB | 无状态转发，不缓存消息 |
